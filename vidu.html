A bit of semantics is in order. The thing most people dub a font is more properly a typeface. Technically, a font is a particular typeface at a particular size
	with a specific set of decorations (underlining, italic, and so on). The distinction is honestly not that important in a digital setting. You don’t explicitly set the font in CSS.
	You determine the font family (which is essentially a typeface), and then you modify its characteristics (creating a font as purists
	would think of it). Still, when I’m referring to the thing that most people call a font (a file in the operating system that describes the appearance of an
	alphabet set), I use the familiar term font
